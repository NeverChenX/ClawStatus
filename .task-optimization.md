## 二、内存优化建议

### 2.1 `_daily_tokens_cache` 内存占用优化

**问题**：
- 当前缓存所有扫描过的 session 文件的 `fileState`、`fileContrib`、`fileContribActive`、`fileContribPassive`
- 对于大量 session 文件（几百个），内存占用会持续增长
- 缓存的天数窗口是 30 天，数据量较大

**优化建议**：
1. **限制缓存文件数**：只保留最近访问/修改的 N 个文件状态（如 200 个）
2. **缩短默认天数**：从 30 天改为 15 天（已经在 UI 显示"最近15天"）
3. **定期清理过期条目**：删除超过窗口时间的 `dayMap` key 和对应的 `fileContrib` 条目
4. **使用弱引用或 LRU 缓存**：当内存压力大时自动淘汰旧数据

**实现要点**：
- 在 `_collect_daily_token_series()` 中添加 LRU 逻辑
- 限制 `fileState` 和 `fileContrib` 的条目数上限
- 添加配置项 `MAX_CACHED_FILES = 200`

**文件位置**：
- `/home/Neverchen/ClawStatus/clawstatus.py:842-1074`（`_collect_daily_token_series`）

### 2.2 缓存大小上限控制

**问题**：
- `_dash_cache`、`_models_cache`、`_memory_cache` 无大小限制
- 理论上可能随着数据增长无限膨胀（虽然目前未发生）

**优化建议**：
1. 添加全局配置：`MAX_CACHE_SIZE_MB = 50`
2. 在缓存更新时检查大小，超限时清空或部分淘汰
3. 使用 `sys.getsizeof()` 或 `pympler` 估算缓存大小

**实现要点**：
- 在 `_invalidate_dashboard_cache()` 中添加大小检查
- 或者使用 `functools.lru_cache` 包装函数

**文件位置**：
- `/home/Neverchen/ClawStatus/clawstatus.py:458-460`（`_invalidate_dashboard_cache`）

### 2.3 减少重复文件扫描

**问题**：
- `_list_session_jsonl_files()` 用 `glob.glob()` 扫描所有 session 文件
- `_build_session_file_passive_map()` 也扫描 `sessions.json` 文件
- 两者可能有重复工作

**优化建议**：
1. 合并扫描逻辑，一次扫描同时收集文件列表和 passive 标记
2. 缓存 `sessions.json` 的解析结果，避免每次重新读取
3. 使用文件 mtime 判断是否需要重新解析

**实现要点**：
- 在 `_daily_tokens_cache` 中添加 `passiveMapCache`
- 只在 `sessions.json` mtime 变化时重新解析

**文件位置**：
- `/home/Neverchen/ClawStatus/clawstatus.py:752-771`（`_list_session_jsonl_files`）
- `/home/Neverchen/ClawStatus/clawstatus.py:774-839`（`_build_session_file_passive_map`）

### 2.4 配置文件解析缓存

**问题**：
- `_safe_read_json(OPENCLAW_CONFIG)` 在多个函数中重复调用
- 每次都重新解析 JSON（虽然有文件系统缓存）
- 配置文件不常变化，可以缓存

**优化建议**：
1. 添加 `_config_cache` 和 `_config_cache_mtime`
2. 检查文件 mtime，只在变化时重新解析
3. 或者使用 `watchdog` 库监听文件变化自动失效缓存

**实现要点**：
- 在 `_safe_read_json()` 或专门的 `_read_openclaw_config()` 中添加缓存
- 使用 `os.stat(path).st_mtime` 检查是否需要重新读取

**文件位置**：
- `/home/Neverchen/ClawStatus/clawstatus.py:94-101`（`_safe_read_json`）

### 2.5 后台刷新线程优化

**问题**：
- `_start_dashboard_refresh()` 在后台线程中定期刷新所有数据
- 即使前端没有请求，也在消耗 CPU 和内存

**优化建议**：
1. 添加"空闲检测"：如果超过 N 分钟无请求，暂停后台刷新
2. 使用增量刷新：只刷新变化的部分
3. 添加配置项控制后台刷新开关

**实现要点**：
- 记录最后请求时间 `_last_request_ts`
- 在后台线程中检查，超时则 sleep 更长时间

**文件位置**：
- `/home/Neverchen/ClawStatus/clawstatus.py:1965-1992`（`_start_dashboard_refresh`）
